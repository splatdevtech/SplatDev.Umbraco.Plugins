WhatsApp Order Tracking — Configure

Send order status and shipment tracking notifications to customers via WhatsApp Business Cloud API.

WhatsApp Business API Credentials
From Meta App Dashboard → WhatsApp → API Setup.
Long-lived System User Access Token with whatsapp_business_messaging permission.
Used to validate X-Hub-Signature-256 on incoming webhooks.
Connected — Business: SplatDev Ltda
Notification Triggers

Enable WhatsApp notifications for each order lifecycle event. Each event maps to a pre-approved message template.

Must match the language of your approved templates in Meta Business Suite.
Message Templates

Map nopCommerce order events to approved WhatsApp template names. Templates must be created and approved in Meta Business Suite → WhatsApp Manager → Message Templates.

Webhook (Delivery Status Callbacks)

Configure a webhook to receive delivery status updates from WhatsApp (sent, delivered, read, failed). Register this URL in Meta App Dashboard → WhatsApp → Configuration → Webhook URL.

https://yourstore.com/Plugins/WhatsAppOrderTracking/Webhook
This URL is auto-generated. Set it in Meta App Dashboard → WhatsApp → Configuration.
Meta sends this value in the hub.verify_token param during webhook registration.

Model / Property Annotations

Section 1 — API Credentials:

  • WhatsAppOrderTrackingSettings.PhoneNumberId — [NopResourceDisplayName("Plugins.Misc.WhatsAppOrderTracking.PhoneNumberId")] string
  • WhatsAppOrderTrackingSettings.AccessToken — password field, encrypted via IEncryptionService
  • WhatsAppOrderTrackingSettings.AppId — string
  • WhatsAppOrderTrackingSettings.AppSecret — password field, encrypted

Section 2 — Notification Triggers:

  • .NotifyOnOrderConfirmed, .NotifyOnOrderProcessing, .NotifyOnShipmentSent, .NotifyOnShipmentDelivered, .NotifyOnOrderCancelled, .NotifyOnRefund — bool
  • .DefaultLanguageCode — string, default "pt_BR"

Section 3 — Message Templates:

  • .TemplateOrderConfirmed, .TemplateShipmentSent, .TemplateShipmentDelivered, .TemplateOrderCancelled, .TemplateRefundIssued — string (template name in Meta)
  • POST /Admin/WhatsAppOrderTracking/ValidateTemplates — checks templates against Meta Graph API

Section 4 — Webhook:

  • .WebhookVerifyToken — string
  • GET/POST /Plugins/WhatsAppOrderTracking/Webhook — public endpoint, no auth; validates X-Hub-Signature-256

Section 5 — Tracking URLs:

  • .DefaultTrackingUrlTemplate — string, {TRACKING} placeholder
  • WhatsAppCarrierUrl entity: CarrierName (string), TrackingUrlTemplate (string) — stored in WhatsAppCarrierUrl table
  • CRUD: /Admin/WhatsAppOrderTracking/CarrierUrlAdd, /CarrierUrlDelete

Section 6 — Message Log:

  • WhatsAppMessageLog entity: OrderId, CustomerId, PhoneNumber, Event, TemplateName, WhatsAppMessageId (wamid), DeliveryStatus (sent/delivered/read/failed), ErrorMessage, CreatedOnUtc, UpdatedOnUtc
  • GET /Admin/WhatsAppOrderTracking/GetLogs?event=&status=&page=1&pageSize=50 → PagedList<WhatsAppMessageLog>

Section 7 — Test Message:

  • POST /Admin/WhatsAppOrderTracking/SendTest — params: phoneNumber, templateName

nopCommerce Event Consumers:

  • IConsumer<OrderStatusChangedEvent> — fires for Confirmed, Processing, Cancelled
  • IConsumer<ShipmentSentEvent> — fires with tracking number + carrier
  • IConsumer<ShipmentDeliveredEvent> — fires on delivery
  • IConsumer<OrderRefundedEvent> — fires on refund

WhatsApp Cloud API:

  • POST https://graph.facebook.com/v21.0/{PhoneNumberId}/messages
  • Auth: Bearer {AccessToken}
  • Category: UTILITY (transactional, not marketing)
