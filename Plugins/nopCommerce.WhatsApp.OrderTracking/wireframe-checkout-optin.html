Model / Property Annotations — Public Views

View 1 — Checkout Opt-in Widget:

  • Rendered as a partial view in the checkout confirm step: Views/WhatsAppOrderTracking/_CheckoutOptIn.cshtml
  • Widget zone: op_checkout_confirm_top or order_summary_content_after
  • Model: WhatsAppOptInModel — OptIn (bool), PhoneNumber (string, with country code)
  • On order placed: stored as GenericAttribute on Customer: "WhatsAppOptIn" (bool) + "WhatsAppPhoneNumber" (string)

View 2 — Order Completed:

  • Widget zone: orderdetails_page_afterproducts or checkout_completed_top
  • Conditionally rendered when WhatsAppOptIn == true on the customer

View 3 — Order Details / Tracking Timeline:

  • Widget zone: orderdetails_page_afterproducts
  • Model: WhatsAppTrackingTimelineModel — OrderId, PhoneNumber, Messages: List<WhatsAppTimelineItem>
  • WhatsAppTimelineItem: Event, SentAt, MessagePreview, DeliveryStatus, TrackingUrl
  • Data source: WhatsAppMessageLog table filtered by OrderId

View 4 — My Account Preferences:

  • Route: /customer/whatsapp-preferences — added via IConsumer<CustomerNavigationEvent>
  • Model: WhatsAppPreferencesModel — GlobalOptIn (bool), PhoneNumber, NotifyConfirmed, NotifyShipped, NotifyDelivered, NotifyCancelled
  • Stored as GenericAttribute on Customer entity

Phone Number Handling:

  • Store in E.164 format: +5511999887766
  • Display with formatting: +55 11 99988-7766
  • Validate via regex: ^\+[1-9]\d{1,14}$ (ITU-T E.164)
  • Pre-fill from billing address phone if available
